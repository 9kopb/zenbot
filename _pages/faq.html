What Zenbot is

Zenbot is an online natural language processing service and chat bot hosting.

What Zenbot can do

Zenbot enables you to create Natural Language driven user interface for your application, website or messenger chat bot. So that you can create Siri-like complex dialog interface for any type of application or website and accept user's text or even voice input.

How does it work

Zenbot matches each text request through a set of currently active patterns in your Botscript. Matched pattern may activate a set of actions in the context which can perform some stuff. Then Zenbot returns one of the text output corresponding this context, so the user is walking through natural language dialog.

How much does it cost

Zenbot is free even for commercial use. No rate limits as well.

How to start

To create a new bot you have to create a Botscript which contains all data about dialog and actions Zenbot must perform.

What I need to know before

All you need to start - is to read about Botscript and look at some of samples. You do not have to have any skills in programming or artificial intelligence discipline.

How to create Botscript

Botscript is a simple XML formatted file. You can create it with any text editor you wish. For exaple Sublime Text or Notepad.

How to upload Botscript to Zenbot

Just got to the web console and create a new bot. Then you will be prompted to upload you Botscript file.

How to use Zenbot from my application or website

Zenbot provides a REST API to allow you make simple HTTP calls to integrate you application or website with your bot.

How to integrate Zenbot with my Facebook, Slack or Telegram bot

Zenbot has already integrated with such messenger platforms like Facebook, Slack and Telegram. So, all you need to bind your Zenbot bot to these messengers - is to provide your credentials in the web console.

What are the differences between Zenbot and Wit.ai

There are a three major differences between Zenbot and Wit.ai:

  • Zenbot implements a full stack of NLP techniques, not only user's intent resolution.

    This means that you do not have to implement a user-to-machine conversation logic on your side. Zenbot proceeds each request, manages contexts, performs actions, evaluates scripts, collects and stores variables and generates text output. Thus the client system just sends a text request and receives a complete response.

  • Zenbot is based on the pattern matching mechanism instead of deep learning "magic".

    Pattern matching provides a NLP flexibility together with a full control of the dialog system. Thus Zenbot takes already human prepared data and works as expected.

  • Zenbot doesn't provide any web interface for dialog structure designing.

    Such web interfaces often bring more confusion than a profit if you have a deal with complex dialogs. Thus Zenbot defines a simple XML file format where you can describe such dialog and manage it as a source code for your bot.
